Get Started with Blocks AI.

Implementing Live Blocks AI into your web-app is easy. There are two simple steps. We are going to bring the core functionaliy. This article will setup your BlocksAI Web Mapper, your placement script and your checkBlock.

Web Mapper.

The Web Mapper in Blocks AI is embedded into every Blocks-AI capable app. It sets the studs ready to communicate with the Blocks AI framework. To get started, place this at the very top of your <body> element.

<script src="https://stella.hs.vc/blocks/blocks.js"></script>

Placement.

The placement is a script that parses the results and places it into your UI or embeds it. This is only needed on the front-end. place this at the very top of your <body> element, but it needs to be after The Mapper.

<script>
function placement(results){
// Enter what happens after everything is processed. Use "results" to reference the answer.
}
</script>

checkBlock.

Add the checkBlock to run *after* your site loads, it will query whatever you put into it, as long as the object is a string. Using this, you will be able use assistant like terms to communicate with Blocks AI. This one is the easiest.

checkBlock(string);

Errors and Codes.

checkBlock(string) is not defined.

The checkBlock was run too early, before the Web Mapper could do its work. Make sure to run it after the page loads, rather than before. Use onload instead of your current workaround.

Cannot read property 'split' of undefined at XMLHttpRequest.Http.onreadystatechange

This error is only present when there is no auto-generated results for what you just searched, it should be disregarded.

DevTools failed to parse SourceMap: https://stella.hs.vc/blocks/math.min.map

Just a general usage error, full functionality will still be present.
